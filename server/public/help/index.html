How can we help?

Everything you need to get MCStatsBot running on your server — setup, report sections, commands, and answers to common questions.

Getting started
1

Download and install the plugin

Grab the latest MCStatsBot-1.0.0.jar from the releases page or SpigotMC and drop it into your server's plugins/ folder.

2

Restart your server

Start or restart your Minecraft server. The plugin will register itself and print a setup URL to the console.

3

Open the setup URL

Find the line in your console that looks like [MCStatsBot] Setup: https://auth.mcstatsbot.tech/setup?... and open it in a browser.

4

Sign in with Discord

Log in with the Discord account that has admin access to the server where you want reports to appear.

5

Pick your channel, language and sections

Select which Discord server and channel should receive reports, what language to use, which sections to include, and what time the daily report should be sent.

6

Done

That's it. The next report will land in your channel at the time you scheduled. No restarts needed.

Report sections

Each daily report is made up of sections. You choose which ones to include during setup. Overview is always included.

Always on Overview
Active players, total playtime, chat count, and player of the day.
Playtime
Top 10 playtime ranking, earliest player online, latest player, longest session.
Combat
Kill and death counts, streaks, PvP kills, mob breakdown, death causes.
Mining
Blocks broken, placed and crafted — with top miners, builders and crafters.
Exploration
Total distance walked, new chunks discovered, Nether and End portal usage.
Achievements
Achievements unlocked today and top XP earners.
Fun Facts
Auto-generated highlights from the day — distance comparisons, pillar heights, and more.
Records
Comparison to yesterday, 7-day activity trends, and all-time leaderboards.
Events
Notable events during the day — dragon kills, first joins, and more.
Commands
Minecraft
/mcstats reload Reloads the plugin configuration from disk. mcstatsbot.admin
Discord
/stats Triggers a report immediately, without waiting for the scheduled time.
Configuration

Most settings are managed through the web dashboard. The only things you might want to change in plugins/MCStatsBot/config.yml directly:

language
Sets the report language. Overridden by the dashboard setting.
en / de
data-retention-days
How many days of stats to keep. Default is 90.
90
afk-timeout-minutes
Minutes without movement before a player counts as AFK and stops accumulating playtime.
5
Common questions
The setup URL doesn't appear in my console.
Make sure the plugin loaded correctly — check your console for any error messages from MCStatsBot. If the plugin loaded but no URL appeared, your server might be having trouble reaching auth.mcstatsbot.tech. Check that outbound HTTPS (port 443) is not blocked by a firewall.
I don't see my Discord server in the setup list.
The bot needs to be in your Discord server before it shows up. During setup, you'll see an invite link — click it to add the bot to your server, then refresh the page. Make sure you're logged in with a Discord account that has Manage Server permissions.
The report is missing some sections.
Sections only show data when there's something to report. Combat won't appear if no one fought anything that day. Mining won't appear if no blocks were broken. If a section consistently doesn't show up, check that the relevant tracking options are enabled in config.yml.
The report is in the wrong language.
Go through the setup flow again — there's a language selector (EN / DE) before you save. The language set there controls the report language and overrides the plugin config. Run /mcstats reload after changing config.yml if you edit it manually.
Players are showing as AFK too quickly / too slowly.
Adjust the afk-timeout-minutes setting in plugins/MCStatsBot/config.yml and run /mcstats reload. The default is 5 minutes without movement.
Can I run reports from multiple Minecraft servers to the same Discord?
Yes. Install the plugin on each server and complete setup for each one separately. Each server gets its own report, and you can send them to the same channel or different channels — your choice.
Can I trigger a report manually?
Yes — use the /stats slash command in your Discord server. The bot will generate and post a report immediately, regardless of the scheduled time.
How do I remove MCStatsBot?
Remove the JAR from plugins/ and restart your server. This immediately stops all data collection. If you want your stored stats deleted from the central server, open an issue on GitHub.
I want to host the central server myself.
The full server code is on GitHub. See SELF_HOSTING.md for the complete setup guide.
